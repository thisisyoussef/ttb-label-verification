## Winning profile so far

- model: `gemini-2.5-flash-lite`
- request profile:
  - raster labels default to `mediaResolution=low`
  - PDF labels default to `mediaResolution=medium`
  - Gemini 2.5 Flash-family models default to `thinkingBudget=0` unless explicitly overridden
  - slimmer `cloud-cross-provider-v2` prompt reduces prompt tokens, but was not kept as the active runtime profile because it did not change the route-level conclusion
- observed service tier on successful runs: `standard`
- current reality:
  - best Gemini extraction average is about `4.62 s`
  - best full-review result on a permissive `6000 ms` run is about `4.95 s`
  - shipped full-review timeout default at `5000 ms` yields about `4.66 s` average with `13/20` success on the checked-in slice
  - the story does not prove the `<= 4000 ms` route target, so the public contract stays at `5000`

## Privacy and trace notes

- OpenAI remains `store: false`.
- Gemini remains inline-only; no Files API and no explicit caching were introduced.
- Internal latency metadata now records only provider name, attempt, service-tier header, and token counts.
- No raw label text, filenames, or application field values were added to timing artifacts.
- The 20-image benchmark slice is now checked into the repo as a synthetic internal corpus; it is no longer local-only under `.tmp`.

## Regressions

- None in the focused deterministic tests.
- The checked-in `GEMINI_TIMEOUT_MS=3000` default still hard-fails the 20-case full-review slice.
- Requested Gemini priority tier was not observed as granted on the successful runs collected here.

## Follow-up

- `TTB-209` closes with the measured non-cutover decision: keep `latencyBudgetMs: 5000`, keep `gemini-2.5-flash-lite`, keep priority disabled by default, and raise the checked-in Gemini timeout default to `5000 ms`.
- Any future sub-`4000 ms` push needs a new story and new measured proof before changing the public contract.
- Investigate additional latency levers only if they preserve the no-persistence posture and do not materially reduce warning-text fidelity.
