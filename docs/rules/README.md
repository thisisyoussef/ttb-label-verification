# Rules

This folder holds compliance-rule traceability for the project.

## Canonical index

- `RULE_SOURCE_INDEX.md` is the repo-level map from validator rule to authoritative source and uncertainty policy.

## Update rules

- Update the index when a validator, severity mapping, or beverage applicability rule changes.
- Story-level details belong in `docs/specs/<story-id>/rule-source-map.md`.
- If a rule is still planned rather than implemented, mark it clearly instead of implying it already exists.
