# Specs

Use `docs/specs/<story-id>/` when a task needs written planning or durable story docs.

Suggested lightweight packet:

- `feature-spec.md`
- `technical-plan.md`
- `task-breakdown.md`

Docs are optional for small, clear tasks.

Use:

- `docs/specs/FULL_PRODUCT_SPEC.md` for product-wide direction
- `docs/specs/PROJECT_STORY_INDEX.md` for the story queue
- `docs/backlog/codex-handoffs/<story-id>.md` for approved UI handoffs

`story-packet.md` is allowed for planning, but active work only needs a checked-in packet when the task actually benefits from one.
