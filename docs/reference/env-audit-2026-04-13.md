# Env Audit

Date: 2026-04-13

## Scope

Audit of real local `.env` files under `/Users/youss/Development/gauntlet`, excluding examples, templates, `node_modules`, build output, and git metadata.

## Findings

- Real env files inspected: `50`
- `OPENAI_API_KEY` exists in multiple local gauntlet env files.
- `ANTHROPIC_API_KEY` also exists locally, but this product does not require it.
- Optional tracing keys exist locally for both LangSmith and Langfuse.
- No current local evidence was found for Gemini, Supabase, Upstash Redis, or Cloudflare keys in the audited real env set.

## Product decision

### Required now

- `OPENAI_API_KEY`
- `OPENAI_MODEL`
- `OPENAI_VISION_MODEL`
- `OPENAI_STORE=false`
- `PORT`

### Optional local AI workflow

- `LANGSMITH_API_KEY`
- `LANGFUSE_PUBLIC_KEY`
- `LANGFUSE_SECRET_KEY`

### Optional local tooling only

- `STITCH_API_KEY`
- `STITCH_PROJECT_ID` (`3197911668966401642` for the canonical `TTB Label Verification System` project)
- `STITCH_FLOW_MODE` (`claude-direct` is now the workspace default)

The repo now prewires LangSmith for trace-driven development by bootstrapping `LANGSMITH_API_KEY`, `LANGSMITH_PROJECT`, and `LANGSMITH_TRACING=false` into the ignored repo-local `.env`. Bootstrap resolves the API key from either `LANGSMITH_API_KEY` or legacy `LANGCHAIN_API_KEY` in the local gauntlet env inventory.

Keep tracing off until there is a concrete LLM or agentic story that needs it. The MVP proof of concept does not require tracing to ship, and tracing must stay local-only against approved fixtures or sanitized inputs.

## What the user still needs to provide

- No new third-party API key is required to finish the current product spec set.
- For implementation, the only required live model credential is an existing `OPENAI_API_KEY`, which already exists elsewhere in the local gauntlet env inventory.
- The repo now includes `npm run env:bootstrap`, which creates or refreshes an ignored repo-local `.env` from that local gauntlet env inventory without printing the keys.
- The local server now loads repo `.env` and `.env.local` automatically outside tests, so a bootstrapped `.env` is enough for local runtime work.
- LangSmith trace-driven development is now also wired through the ignored repo-local `.env` plus `npm run langsmith:smoke`.
- The optional automated Stitch workflow can use either a local-only `STITCH_API_KEY` or the ignored project-local Stitch MCP config in `.mcp.json` / `.cursor/mcp.json`.
- The workspace now defaults to `STITCH_FLOW_MODE=claude-direct`; automated Stitch remains available via `STITCH_FLOW_MODE=automated`, and manual Comet remains available via `STITCH_FLOW_MODE=manual`. `STITCH_API_KEY` remains tooling config, not app runtime config.
- For Railway, the same runtime variables need to exist in both `staging` and `production` environments.
- GitHub Actions deploys now require a Railway token secret in GitHub. The current harness uses `RAILWAY_API_TOKEN`.

## Security note

This audit intentionally records key presence and applicability, not secret values or source-file contents.
