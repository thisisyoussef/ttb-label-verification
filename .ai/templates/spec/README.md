# Spec Templates

Standard feature work should produce a packet under `docs/specs/<story-id>/`.

Recommended files:

- `constitution-check.md`
- `feature-spec.md`
- `technical-plan.md`
- `task-breakdown.md`
- `ui-component-spec.md` when UI scope is material; this is the per-feature design doc and should align to `docs/design/MASTER_DESIGN.md`
- `evidence-contract.md` when the evidence payload or result details change
- `rule-source-map.md` when validator behavior or citations change
- `privacy-checklist.md` when uploads or model calls change
- `performance-budget.md` when the single-label critical path changes
- `eval-brief.md` when AI behavior, routing, grading, or prompt behavior changes
