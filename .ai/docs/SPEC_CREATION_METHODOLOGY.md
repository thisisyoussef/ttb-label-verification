# Spec Method

Specs are optional in the current workflow.

Use `docs/specs/<story-id>/` when the work is large, ambiguous, cross-cutting, risky, or when the user asks for written planning artifacts.

Default lightweight packet when you do need one:

- `feature-spec.md`
- `technical-plan.md`
- `task-breakdown.md`

Story ids:

- `TTB-WF-*` for workflow or harness work
- `TTB-EVAL-*` for eval-only work
- `TTB-1xx` to `TTB-4xx` for product stories
`story-packet.md` is fine for planning, but active work should have enough checked-in detail to be recoverable from the repo when docs are actually needed.
