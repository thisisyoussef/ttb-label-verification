# Story Lookup

Use this before meaningful implementation.

1. Read `AGENTS.md` and SSOT.
2. Confirm the story and lane from checked-in docs.
3. Check the current branch.
4. If the current checkout belongs to another story, switch to a fresh story branch. If it is dirty and you need isolation, create a sibling worktree from `origin/main`.
5. Record the branch in `docs/process/BRANCH_TRACKER.md`.
6. Read the nearest code, docs, and tests.
7. Write down the blast radius, implementation implications, and test plan before coding.
